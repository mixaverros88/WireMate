Overview

#overview

The WireMock Admin API is a RESTful HTTP interface that lets you control a running WireMock instance without restarting it. It supports the full lifecycle of stub management, request verification, stateful scenario control, and server configuration. The API follows OpenAPI 3.0 and an interactive Swagger UI is bundled with every WireMock instance.

Base URL & Access

#base-url

WireMock defaults to port 8080. Every Admin API endpoint is under the /__admin/ path prefix.

Base URL 
http://<host>:<port>/__admin/
Quick Sanity Check 
# Verify WireMock is running and list all stubs
curl http://localhost:8080/__admin/mappings

# Check health
curl http://localhost:8080/__admin/health

# View Swagger UI
open http://localhost:8080/__admin/docs/
ℹ️
Security Note
The Admin API is unauthenticated by default. In production-like environments consider using --admin-api-require-https-client-cert or fronting WireMock with a reverse proxy to restrict access.

Conventions

#conventions

All request bodies are JSON. All responses are JSON. The Content-Type: application/json header must be sent with any body. Path parameters are shown as {id}. UUIDs are used as resource identifiers throughout.

HTTP Method Semantics
MethodSemanticsIdempotent
GET Retrieve resource(s) Yes
POST Create new resource or execute action No
PUT Replace existing resource by ID Yes
DELETE Remove resource or collection Yes

Stub Mappings — CRUD

#mappings

Stub mappings are the core building block of WireMock. Each mapping pairs a request matcher with a response definition.

GET /__admin/mappings Get all stub mappings ▾

Returns all currently registered stub mappings. Supports pagination via limit and offset query parameters. Useful as a quick health check — if WireMock is up, you'll get a valid JSON response.

Query Parameters
NameTypeRequiredDescription
limit integer Optional Maximum number of mappings to return
offset integer Optional Offset for pagination (zero-based)
cURL
curl http://localhost:8080/__admin/mappings
Response 200
{
  "mappings": [
    {
      "id": "e5e3a42e-f00f-4a2c-8a6b-1c2d3e4f5a6b",
      "priority": 1,
      "request": {
        "method": "GET",
        "url": "/api/users"
      },
      "response": {
        "status": 200,
        "body": "[{\"id\":1,\"name\":\"Alice\"}]",
        "headers": { "Content-Type": "application/json" }
      }
    }
  ],
  "meta": { "total": 1 }
}
Responses
200Paginated list of stub mappings
POST /__admin/mappings Create a new stub mapping ▾

Creates a new stub mapping. Returns the created mapping with its server-assigned UUID. A single mapping object or an array of mappings can be submitted.

cURL — Simple GET stub
curl -X POST http://localhost:8080/__admin/mappings \
  -H "Content-Type: application/json" \
  -d '{
    "request": {
      "method": "GET",
      "url": "/api/users",
      "headers": {
        "Content-Type": { "equalTo": "application/json" }
      }
    },
    "response": {
      "status": 200,
      "body": "[{\"id\":1,\"name\":\"Alice\"}]",
      "headers": { "Content-Type": "application/json" }
    }
  }'
cURL — With query params & priority
curl -X POST http://localhost:8080/__admin/mappings \
  -H "Content-Type: application/json" \
  -d '{
    "priority": 5,
    "request": {
      "method": "GET",
      "urlPath": "/greet",
      "queryParameters": {
        "name": { "equalTo": "Alice" }
      }
    },
    "response": {
      "status": 200,
      "body": "Hello, Alice!"
    }
  }'
Responses
201Stub created — returns the mapping with ID
400Invalid request body
DELETE /__admin/mappings Delete all stub mappings ▾

Removes all currently registered stub mappings. Equivalent to POST /__admin/mappings/reset. Persistent stubs loaded from disk on startup are also removed from the in-memory registry but not deleted from disk.

cURL
curl -X DELETE http://localhost:8080/__admin/mappings
Responses
200All mappings removed
POST /__admin/mappings/reset Reset to default stub mappings ▾

Resets all stub mappings to their startup defaults, reloading stubs from the mappings directory on disk. In-memory stubs added via the API since startup are discarded.

cURL
curl -X POST http://localhost:8080/__admin/mappings/reset
Responses
200Reset successful
POST /__admin/mappings/save Persist all stub mappings to disk ▾

Saves all currently registered stub mappings to the mappings/ directory as JSON files, making them persistent across server restarts.

cURL
curl -X POST http://localhost:8080/__admin/mappings/save
GET /__admin/mappings/{id} Get a single stub mapping ▾

Retrieves a specific stub mapping by its UUID.

Path Parameters
NameTypeRequiredDescription
id string (UUID) Required The UUID of the stub mapping
cURL
curl http://localhost:8080/__admin/mappings/e5e3a42e-f00f-4a2c-8a6b-1c2d3e4f5a6b
Responses
200The mapping object
404Mapping not found
PUT /__admin/mappings/{id} Update a stub mapping ▾

Replaces an existing stub mapping by ID with the provided definition. The full mapping object must be supplied — this is a full replacement, not a patch.

cURL
curl -X PUT http://localhost:8080/__admin/mappings/e5e3a42e-f00f-4a2c-8a6b-1c2d3e4f5a6b \
  -H "Content-Type: application/json" \
  -d '{
    "request": { "method": "GET", "url": "/api/users" },
    "response": { "status": 200, "body": "updated" }
  }'
Responses
200Updated mapping returned
404Mapping not found
DELETE /__admin/mappings/{id} Delete a stub mapping ▾

Removes a single stub mapping by its UUID.

cURL
curl -X DELETE http://localhost:8080/__admin/mappings/e5e3a42e-f00f-4a2c-8a6b-1c2d3e4f5a6b
Responses
200Deleted
404Not found

Stub Mappings — Metadata Queries

#mappings-find

Stubs can be tagged with arbitrary metadata. These endpoints let you find or bulk-delete stubs by matching on that metadata.

POST /__admin/mappings/find-by-metadata Find stubs matching metadata criteria ▾

Returns all stubs whose metadata field satisfies the provided matcher expression. Supports all standard WireMock matchers (equalTo, contains, matches, etc.).

cURL
curl -X POST http://localhost:8080/__admin/mappings/find-by-metadata \
  -H "Content-Type: application/json" \
  -d '{
    "matchesJsonPath": "$[?(@.team == 'backend')]"
  }'
POST /__admin/mappings/remove-by-metadata Remove stubs matching metadata criteria ▾

Deletes all stubs whose metadata satisfies the provided matcher. Useful for cleanup after a test suite that tags its stubs with a test-run ID.

cURL
curl -X POST http://localhost:8080/__admin/mappings/remove-by-metadata \
  -H "Content-Type: application/json" \
  -d '{ "equalTo": { "runId": "test-run-42" } }'

Stub Mappings — Unmatched

#mappings-unmatched

Identify stubs that have never been triggered. Useful for dead-code detection in your mock layer.

GET /__admin/mappings/unmatched Get stubs that matched no requests ▾

Returns all stub mappings that have not matched any incoming request.

cURL
curl http://localhost:8080/__admin/mappings/unmatched
DELETE /__admin/mappings/unmatched Remove stubs that matched no requests ▾

Deletes all stubs that have not yet matched any request.

cURL
curl -X DELETE http://localhost:8080/__admin/mappings/unmatched

Request Journal

#requests

WireMock records every incoming request in an in-memory journal. These endpoints expose that journal for inspection and verification in test scenarios.

GET /__admin/requests Get all logged requests ▾

Returns all requests recorded in the journal, newest first. Supports limit and since query parameters.

cURL
curl http://localhost:8080/__admin/requests
Response 200
{
  "requests": [
    {
      "id": "a1b2c3d4-...",
      "request": {
        "url": "/api/users",
        "method": "GET",
        "headers": { "Accept": "application/json" }
      },
      "responseDefinition": { "status": 200 },
      "wasMatched": true
    }
  ],
  "meta": { "total": 1 }
}
DELETE /__admin/requests Clear the request journal ▾

Wipes the entire request journal. Call this at the start of each test to ensure a clean slate.

cURL
curl -X DELETE http://localhost:8080/__admin/requests
GET /__admin/requests/{id} Get a specific logged request ▾

Retrieves a single logged request by UUID.

cURL
curl http://localhost:8080/__admin/requests/a1b2c3d4-1234-5678-abcd-000000000001
DELETE /__admin/requests/{id} Delete a specific logged request ▾

Removes a single request from the journal by UUID.

cURL
curl -X DELETE http://localhost:8080/__admin/requests/a1b2c3d4-1234-5678-abcd-000000000001
GET /__admin/requests/unmatched Get unmatched incoming requests ▾

Returns all requests that arrived but did not match any stub. These are the requests that caused WireMock to return a 404. Useful for debugging gaps in your stub configuration.

cURL
curl http://localhost:8080/__admin/requests/unmatched

Request Journal — Find & Count

#requests-find

Query the request journal using the same matcher DSL used in stub definitions. Supports all URL, header, body, and query parameter matchers.

POST /__admin/requests/find Find requests matching a pattern ▾

Returns all requests in the journal that satisfy the given request pattern. The pattern uses the same matcher format as stub request definitions.

cURL
curl -X POST http://localhost:8080/__admin/requests/find \
  -H "Content-Type: application/json" \
  -d '{
    "method": "POST",
    "urlPath": "/api/orders",
    "bodyPatterns": [
      { "contains": "\"status\":\"NEW\"" }
    ]
  }'
POST /__admin/requests/count Count requests matching a pattern ▾

Returns the count of requests that match the given pattern. Same request body shape as POST /__admin/requests/find.

cURL
curl -X POST http://localhost:8080/__admin/requests/count \
  -H "Content-Type: application/json" \
  -d '{ "method": "GET", "url": "/api/ping" }'
Response 200
{ "count": 3 }
POST /__admin/requests/remove Remove requests matching a pattern ▾

Deletes from the journal all requests that match the given pattern.

cURL
curl -X POST http://localhost:8080/__admin/requests/remove \
  -H "Content-Type: application/json" \
  -d '{ "method": "GET", "url": "/api/health" }'

Near Misses

#near-misses

Near misses are stubs that almost matched a request. WireMock scores each unmatched request against every stub and returns the closest matches, making it easy to diagnose misconfigured matchers.

GET /__admin/requests/unmatched/near-misses Near misses for unmatched requests ▾

Returns near-miss analysis for every request that wasn't matched by a stub. Each entry shows which stub came closest and why it didn't match.

cURL
curl http://localhost:8080/__admin/requests/unmatched/near-misses
POST /__admin/near-misses/request Near misses for a specific request ▾

Given a request object in the body, returns the stub mappings that most closely matched it. Useful for debugging without needing to fire a real HTTP request.

cURL
curl -X POST http://localhost:8080/__admin/near-misses/request \
  -H "Content-Type: application/json" \
  -d '{
    "method": "DELETE",
    "url": "/api/users/42",
    "headers": { "Authorization": "Bearer token123" }
  }'
POST /__admin/near-misses/request-pattern Near misses for a request pattern ▾

Given a request pattern (matcher), returns the logged requests from the journal that most closely match it. Helps verify that your stub's matcher covers the requests you expect.

cURL
curl -X POST http://localhost:8080/__admin/near-misses/request-pattern \
  -H "Content-Type: application/json" \
  -d '{
    "method": "POST",
    "urlPathPattern": "/api/users/[0-9]+"
  }'

Scenarios

#scenarios

Scenarios enable stateful behaviour. A scenario is a named state machine; stubs can be conditional on the current scenario state and can transition it to a new state on match.

GET /__admin/scenarios Get all scenarios and their current state ▾

Returns all defined scenarios with their current state.

cURL
curl http://localhost:8080/__admin/scenarios
Response 200
{
  "scenarios": [
    {
      "id": "order-flow",
      "name": "order-flow",
      "state": "Started",
      "possibleStates": ["Started", "order-placed", "order-confirmed"]
    }
  ]
}
POST /__admin/scenarios/reset Reset all scenarios to initial state ▾

Resets all scenarios back to Started state. Should be called between test cases to ensure a consistent starting point.

cURL
curl -X POST http://localhost:8080/__admin/scenarios/reset
PUT /__admin/scenarios/{name}/state Set the state of a specific scenario ▾

Forces a specific scenario into a given state without requiring HTTP traffic to trigger the transition. Useful for setting up a test mid-flow.

cURL
curl -X PUT http://localhost:8080/__admin/scenarios/order-flow/state \
  -H "Content-Type: application/json" \
  -d '{ "state": "order-placed" }'

Recordings

#recordings

WireMock can proxy and record real API traffic, saving it as replayable stub mappings. This is the fastest way to bootstrap a mock from an existing service.

POST /__admin/recordings/start Start recording ▾

Starts recording incoming proxied requests as stubs. The request body configures the target URL and recording options.

cURL
curl -X POST http://localhost:8080/__admin/recordings/start \
  -H "Content-Type: application/json" \
  -d '{
    "targetBaseUrl": "https://api.real-service.com",
    "filters": {
      "urlPathPattern": "/api/.*"
    },
    "captureHeaders": {
      "Accept": { "caseInsensitive": true }
    }
  }'
POST /__admin/recordings/stop Stop recording ▾

Stops recording and returns all stub mappings that were generated during the session.

cURL
curl -X POST http://localhost:8080/__admin/recordings/stop
GET /__admin/recordings/status Get recording status ▾

Returns whether recording is currently active.

cURL
curl http://localhost:8080/__admin/recordings/status
Response 200
{ "status": "NeverStarted" }
// Possible values: NeverStarted | Recording | Stopped
POST /__admin/recordings/snapshot Take a recording snapshot ▾

Generates stub mappings from the requests recorded since the last snapshot (or since recording started). Does not stop the recording session — allows incremental snapshotting.

cURL
curl -X POST http://localhost:8080/__admin/recordings/snapshot

Global Settings

#settings

Read and modify server-wide configuration at runtime without restarting.

GET /__admin/settings Get current global settings ▾

Returns the current global configuration of the WireMock instance.

cURL
curl http://localhost:8080/__admin/settings
Response 200
{
  "fixedDelay": 0,
  "delayDistribution": null,
  "extendedCaptureHeadersPatterns": [],
  "webhookMaxThreads": 10
}
POST /__admin/settings Update global settings ▾

Updates global settings. Only the provided fields are changed. Useful for adding a global response delay to simulate latency.

cURL — Add 200ms global delay
curl -X POST http://localhost:8080/__admin/settings \
  -H "Content-Type: application/json" \
  -d '{ "fixedDelay": 200 }'
cURL — Random delay distribution
curl -X POST http://localhost:8080/__admin/settings \
  -H "Content-Type: application/json" \
  -d '{
    "delayDistribution": {
      "type": "uniform",
      "lower": 50,
      "upper": 300
    }
  }'

System

#system

Lifecycle and status endpoints for the WireMock server process itself.

GET /__admin/health Health check ▾

Returns HTTP 200 with body {"status":"healthy"} if the server is running and processing requests normally. Returns 503 if unhealthy.

cURL
curl http://localhost:8080/__admin/health
Response 200
{ "status": "healthy" }
GET /__admin/version Get WireMock version ▾

Returns the version string of the running WireMock instance.

cURL
curl http://localhost:8080/__admin/version
Response 200
{ "version": "3.5.4" }
POST /__admin/reset Full reset — mappings + journal ▾

Resets all stub mappings to their startup defaults AND clears the request journal in one call. The most complete teardown available short of restarting the process.

cURL
curl -X POST http://localhost:8080/__admin/reset
⚠️
Destructive
This endpoint erases all in-memory stubs and all recorded requests. Stubs saved to disk via /__admin/mappings/save are not deleted and will be reloaded on the next reset.
POST /__admin/shutdown Shut down the WireMock server ▾

Gracefully shuts down the WireMock process. Send an empty body or no body. The server will not respond to further requests after this call.

cURL
curl -X POST http://localhost:8080/__admin/shutdown
⚠️
Irreversible
This terminates the WireMock process. It cannot be used in embedded mode (JUnit integration) — use wireMockServer.stop() instead.
